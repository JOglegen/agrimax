🔴 #7: Work Order Field Data Bug

CRITICAL BUG
Problem
When new field created and selected in work order, lab results and soil summary from unrelated fields display instead of correct data.
Root Causes
  • Field ID not being correctly saved in work_orders.field_id
  • Lab results query filtering by wrong column
  • Soil summary pulling from wrong entity
  • Work order field reference stale/null
Solution Steps
1. Add debug logging to renderWoDetail()
console.log("Work Order:", wo); console.log("Field ID:", wo?.field_id); console.log("Lab Results:", DATA.lab_results?.filter(lr => lr.work_order_id === wo?.id));
2. Verify field assignment saves correctly when selected
3. Fix data queries to use correct filters (work_order_id for lab results, field_id for soil summary)
4. Update renderWoDetail to use corrected queries
Estimated Time: 2-3 hours
Priority: CRITICAL - Data integrity issue

🔴 #2: JD Boundary Data Missing

HIGH BUG
Problem
Fields imported from John Deere Operation Center lack boundary/polygon data needed for map display.
Solution
1. Debug JD API response to identify geometry location
console.log("Full JD Response:", jdResponse); console.log("Sample Field:", jdResponse.values?.[0]); console.log("Keys:", Object.keys(jdResponse.values?.[0]));
2. Extract boundary data from response (check: geometry, boundary, polygon, shape, @geometry)
3. Map boundary data to field geojson column
4. Display boundaries on map with L.geoJSON()
Estimated Time: 2 hours
Priority: HIGH - Map functionality

🟠 #13: Password Reset Email

HIGH FEATURE
Problem
Admins cannot send password reset emails to users. Critical for user management.
Solution
async function sendPasswordResetEmail(userId, userEmail) { const { error } = await supa.auth.resetPasswordForEmail(userEmail, { redirectTo: window.location.origin + '/reset-password' }); if(!error) { await supa.from('audit_log').insert({ action: 'password_reset_sent', target_user_id: userId }); toast("Email sent to " + userEmail); } }
Estimated Time: 2 hours
Difficulty: Medium
