   4. Select your AgriMax repository

1. You push HTML files to GitHub
2. Netlify sees changes automatically
3. Netlify builds and deploys your site
4. Your documentation goes live in ~30 seconds
5. No additional configuration needed!

When you update files:
  → Edit HTML locally
  → Commit and push to GitHub
  → Netlify automatically redeploys
  → Changes live in seconds

EXAMPLE - Updating Issue Status:
  1. Open issues.html in text editor
  2. Find issue #7 section
  3. Change: <span class="status-badge">Open</span>
     To:     <span class="status-badge">Done ✓</span>
  4. Save file
  5. git add issues.html
  6. git commit -m "docs: mark issue #7 complete"
  7. git push
  8. Netlify auto-deploys within 30 seconds!

❓ TROUBLESHOOTING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Problem: Netlify not auto-deploying
Solution: 
  1. Check GitHub repository is PUBLIC
  2. Verify Netlify is connected to your GitHub account
  3. Check Netlify dashboard for errors
  4. Try manual trigger: Redeploy in Netlify dashboard

Problem: Pages not showing up
Solution:
  1. Verify HTML files are in root folder (not in subfolders)
  2. Check file names are exactly: index.html, issues.html, fixes.html
  3. Wait 1-2 minutes for deployment to complete
  4. Hard refresh browser (Ctrl+Shift+R or Cmd+Shift+R)

Problem: Broken links between pages
Solution:
  1. Verify all links in HTML files point to correct files
  2. Links should be: href="issues.html" (not full URLs)
  3. Check file names match exactly (case-sensitive on Linux)

Problem: Styling looks wrong
Solution:
  1. Hard refresh browser (clear cache)
  2. Check that CSS is embedded in HTML (it is)
  3. Try different browser
  4. Check mobile/responsive view
